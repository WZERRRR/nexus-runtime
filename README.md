# 🌟 DevCore AI — Predictive Enterprise OS (النظام التشغيلي الاستخباري للمؤسسات)

## 📖 مقدمة عن المشروع
**DevCore AI** هو نظام تشغيل (OS) متطور مبني على الويب، يمثل الجيل القادم من منصات الحوكمة الإدارية والتقنية. صُمم النظام ليكون المركز العصبي للمؤسسات التقنية، حيث يدمج بين إدارة البنية التحتية، حوكمة المشاريع، والذكاء الاصطناعي التنبؤي لضمان استقرار العمليات وتقليل المخاطر التشغيلية.

---

## 🛰️ المكونات الاستخبارية (Intelligence Layer)
يتميز النظام بقدرات فريدة تتجاوز المراقبة التقليدية:
- **تحليل التنبؤ بالفشل (Failure Forecasting):** استخدام خوارزميات إحصائية للتنبؤ باحتمال تعطل العقد أو السيرفرات قبل وقوعه.
- **مؤشرات المخاطر (Risk Indicators):** مراقبة حية للتهديدات الأمنية، محاولات الاختراق، واستقرار النظام.
- **الذكاء التشغيلي (Operational Intelligence):** رصد إشارات الأداء عبر كافة طبقات النظام (Hardware, Network, App).
- **مراقب الذكاء الاصطناعي (AI Monitor):** تتبع استهلاك النماذج مثل Google Gemini وتكاليف الـ Tokens لحظياً.

---

## 🚀 الوحدات التشغيلية الشاملة (System Modules)

### 1. إدارة البنية التحتية والشبكات (Infrastructure)
- **Servers Center:** مراقبة حية لجميع السيرفرات، العقد (Nodes)، والحاويات (Containers).
- **Domains & SSL Gateway:** إدارة كاملة للنطاقات عبر بوابة Nexus Edge مع تتبع دورة حياة شهادات SSL وتجديدها.
- **Web Terminal:** وصول مباشر للطرفية (Terminal) عبر SSH مؤمن للتحكم السريع.
- **Databases:** إدارة قواعد البيانات (MySQL, SQLite) ومراقبة الاستعلامات والأداء.

### 2. حوكمة التطبيقات والمشاريع (Governance)
- **Deployment Pipelines:** نظام CI/CD مدمج لتتبع عمليات النشر والاختبار.
- **Laravel & Frameworks:** واجهات متخصصة لإدارة تطبيقات Laravel (Artisan, Migration, Routes).
- **PM2 Manager:** تحكم كامل في عمليات Node.js، إعادة التشغيل، ومراقبة الذاكرة.
- **Environments Sync:** مزامنة المتغيرات والبيئات (Production, Staging, Dev) بمعايير أمنية عالية.

### 3. العمليات والأمن (Operations & Security)
- **Security Operations Center (SOC):** رصد الهجمات (Brute Force, SQLi) وعزل التهديدات تلقائياً.
- **OTP Gateway:** نظام إدارة رموز التحقق والمصادقة الثنائية.
- **Backups & Recovery:** جدولة النسخ الاحتياطية ونقاط الاستعادة (Restore Points) لضمان استمرارية الأعمال.
- **Payments Integration:** لوحة تحكم مالية متكاملة لربط Stripe و Moyasar وتتبع التدفقات النقدية.

## 🛠️ المكدس التقني المعتمد (Technical Stack)

النظام مبني على معايير Enterprise لضمان القابلية للتوسع:

- **Frontend:** 
  - **React 19:** الواجهة التفاعلية الأحدث.
  - **Vite 6:** محرك البناء فائق السرعة.
  - **Tailwind CSS 4:** نظام التصميم والواجهات العصرية.
  - **Motion (Framer):** التحريكات البصرية السلسة والتفاعلات.
  - **Recharts:** تمثيل البيانات التنبؤية برسوم بيانية متقدمة.

- **Backend (Full-stack):**
  - **Express.js:** خادم الويب ومعالجة الطلبات.
  - **Better-SQLite3:** قاعدة بيانات محلية سريعة جداً لإدارة الـ Runtime والتنبؤات.
  - **SystemInformation:** جلب بيانات العتاد والسيرفرات الحقيقية.
  - **Google GenAI SDK:** التكامل مع نماذج Gemini للذكاء الاصطناعي.

## 📁 هيكلية النظام (Architecture & Folders)

```text
├── server               # محرك الواجهة الخلفية (Backend)
│   ├── database         # إدارة اتصالات وقواعد بيانات النظام (SQLite/MySQL)
│   ├── services         # الخدمات الخلفية (الذكاء التشغيلي، المحاكي، إلخ)
│   └── server.ts        # الملف الرئيسي لتشغيل الخادم والهندسة المركزية
├── src                  # الواجهة الأمامية (Frontend)
│   ├── components       # المكونات القابلة لإعادة الاستخدام (UI/UX)
│   ├── pages            # الوحدات الوظيفية (30+ صفحة تخصصية)
│   ├── services         # خدمات الاتصال بالـ APIs والـ Runtime API
│   └── App.tsx          # الموجه المركزي للنظام
├── runtime              # بيانات التشغيل والملفات المؤقتة لنظام الاستخبارات
├── metadata.json        # بيانات الهوية والوصف للنظام
└── .env.example         # نموذج لمتغيرات البيئة الضرورية
```

## ⚖️ قواعد التشغيل والحوكمة (Operations Rules)
- **RTL & Dark Mode:** النظام يدعم اللغة العربية بشكل كامل مع تصميم مريح للعين في بيئات العمل الطويلة.
- **Persistence First:** الحفاظ على بيانات الجلسات والعمليات عبر SQLite لضمان عدم ضياع التنبؤات عند إعادة التشغيل.
- **Clean Architecture:** التزام صارم بفصل الاهتمامات بين الـ Presentation والـ Logic.
